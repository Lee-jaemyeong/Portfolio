- **성과 요약**:
  - 예약 및 결제 CRUD 설계 및 기능 구현
  - 추천받은 경로, 다른 사용자의 경로대로 일괄 예약 기능 구현

  - 예약에 자동 만료일 부여 → 만료시 다른 사용자도 예약가능
  
**🔹기술스택**
- Spring Boot 2.7.14 / Java 11 / JPA 2.7.14 / RESTful API 2.7.14 / Lombok 1.18.28 / Thymeleaf 3.0.15
- React 18.20.0 / Next.js 15.3.5 / Axios 1.9.0 
- MySQL 8.0.33 / MyBatis 2.3.1

**🔹트러블슈팅**

**1. 예약 날짜 일괄 비활성화 오류**
  
  - 문제: 단일 장소에서 예약한 날짜가 전체 예약 건에 일괄 적용 → 모든 장소의 동일한 날짜가 예약 불가능 처리
  - 분석: 장소(place_id) 조건 없이 날짜만을 기준으로 비활성화 → 모든 장소에 동일한 날짜 비활성화 적용 
  - 해결: 예약 비활성화 로직에서 장소 식별자(place_id) 조건 포함 → JOIN 구문을 통해 장소별 예약 날짜 필터링 → 장소별로 독립된 비활성화 날짜 처리 가능
     
**2. 결제 시 예약 정보 누락**

  - 문제: 결제 진행 시, 예약 정보가 화면에 미출력 혹은 null 값으로 표시되는 문제 발생
  - 분석: 결제 매퍼에서 예약id만 조회하고, 이에 따른 예약 상세 정보를 Join하여 가져오지 않아 필요한 정보가 누락
  - 해결: 결제 매퍼에 예약 테이블과의 JOIN 구문을 추가 → 예약 ID 뿐만 아니라 관련된 예약 상세 정보를 함께 조회하도록 수정
  - 
**3. 결제 취소 및 환불 시 예약 상태 미반영**

     - 문제: 결제 취소 또는 환불을 진행해도 해당 예약 상태가 여전히 "예약 중"으로 유지되는 문제 발생 
     - 분석: 결제 및 예약 상태 간 연결 관계 미설정 → 결제 취소 처리 시 예약 상태 업데이트 실패
     - 해결: 예약 상태와 결제 상태를 Enum 타입으로 정의 → 예약 상태가 취소일 경우에만 결제 취소 또는 환불을 가능하게 하는 로직 구성 → 결제 상태 변경과 예약 상태 변경을 동기화
       
**4. 일괄 예약 시 경로 정보 미조회**

  - 문제: 여러 장소를 포함하는 일괄 예약 시, 경로 정보 조회 실패 → 예약 데이터가 null로 저장되는 오류 발생
  - 분석: 초기 테스트에서는 더미 데이터를 기반으로 경로 ID만 연결하여 기능 확인 → 실제 경로 정보 테이블과의 관계 설정이 누락
  - 해결: 예약 테이블이 경로 정보 테이블과 관계를 맺도록 수정 → JOIN 구문을 통해 일괄 예약 시 각 경로의 정보를 조회할 수 있도록 개선
    
**5. 일괄결제 취소, 환불 오류**

  - 문제: 일괄 결제의 취소 또는 환불 요청 미처리
  - 분석: 일괄 결제는 여러 예약을 경로 ID 기준으로 묶어 총 금액을 계산하는 구조 → 결제 테이블에 예약 ID 저장 불가 → 환불 처리 시 참조할 예약 정보가 미존재
  - 해결: 결제 테이블 중심이 아닌, 각 예약이 결제 ID를 참조하도록 구조 개선 → 환불 로직에서는 동일 결제id를 가지는 예약들을 묶어 처리하도록 변경 → 일괄 결제에 대한 환불도 정상 작동하게 개선
     
**🔹협업 및 소감**
